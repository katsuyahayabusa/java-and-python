Python et Java, Ressemblances et différences

I- Détails sur les langages

Python et Java ont une différence fondamentale au niveau de la conception du code du programme : là où Python met plus l’accent sur la facilité de lecture du code, Java préfère utiliser le moins de ressources possibles pour exécuter son programme. De plus, un code java aura besoin d’être compilé avant, alors que python est interprété directement par l’ordinateur lors de l’exécution du programme.
Les deux langages sont cependant tous deux des langages de haut niveau, donnant une grande liberté d’action aux programmeurs.
Java a été conçu pour permettre aux développeurs de créer des applications logicielles avec des dépendances minimales pour la mise en œuvre, alors que Python est un langage plutôt axé sur la lisibilité du code, et est donc un langage parfait pour les néophytes en programmation. De plus, Java appartient à Oracle, et ne peut donc pas être modifié, alors que Python est open source.
Les 2 langages sont cependant exploitables sur plusieures plateformes : Mobile, Web….

II- Structure du code

    On peut remarquer des différences majeures dans la structure des deux langages:
  1. En codant en java, il faut déclarer le type de chaque variable qu'on utilise (entier, chaîne de caractère, booléen, flottant par exemple)
  2. Java possède une syntaxe assez stricte, plus proche du C++ que du Python: il faut terminer toutes ses commandes par un point virgule (;), et de ce fait, les lignes de code java sont plus longues et plus riches que celles d'un langage typé comme Python
    Python est un langage assez différent de son confrère sur de nombreux points:
  1. C'est un langage de programmation compact, ce qui signifie qu'on a besoin de très peu de mots pour coder un programme
  2. C'est un langage typé dynamiquement: Le type des variables n'a pas besoin d'être déclaré, et est déterminé au cours de la lecture du programme par l'interprète
  3. Python est assez libre concernant la syntaxe: il n'y a pas besoin de terminer ses lignes de code par des accolades, mais le code a quand même besoin d'être indenté pour être exécuté
